# Workforce Docs Admin Support

This folder contains non-canonical support materials that are useful for planning, repo installation, Copilot execution, progress tracking, and merge operations.

## What belongs here
- Copilot install prompts and setup instructions
- master-plan formatting and status templates
- atomic execution wrappers
- progress report templates
- merge patch plans
- reference PDFs
- example HN3T master-plan sections and handoff files

## What does not belong here
Primary architecture/module specifications that define the intended Workforce system-of-record design. Those live one level up in `docs/`.

## Suggested usage
- Use `planning/templates/` when creating new specs and master-plan sections.
- Use `planning/workforce_studio/` for Studio planning examples.
- Use `repo_merge/` when merging this framework into an existing repo.
- Use `ai_comm_studio_support/` for installation prompts and reference material around the AI/Communication/Studio spec pack.

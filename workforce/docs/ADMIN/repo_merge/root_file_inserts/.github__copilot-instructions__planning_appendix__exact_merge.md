## Planning and atomic execution workflow

Use `HN3T_MASTER_PLAN.md` as the planning source of truth for implementation sequencing.

### Execution discipline
- execute one checkbox at a time when using the plan runner workflow
- keep patches minimal and reviewable
- avoid scope creep into neighboring tasks
- preserve tenant isolation, RBAC, and audit safety rules

### Progress reporting
- update `PROGRESS_REPORT.md` after each processed atomic task
- record exact checkbox text
- record plan state and outcome code
- record files touched and tests run

### Safety
- reject plan-execution patches that weaken tenant scope rules
- reject plan-execution patches that bypass RBAC
- reject plan-execution patches that mutate unrelated areas just because the model is "already there"
